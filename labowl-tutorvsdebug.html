Two Approaches to Helping Students

See the difference between traditional tutoring and debugging literacy coaching

The Scenario

A student is working on a Python lab. Their code runs but produces the wrong output. They've been staring at it for 20 minutes and just changed three things at once without knowing which one might fix it.

Traditional Tutoring

Focus: Teaching the content

S

"What's the problem with my code?"

"It's not giving the right answer."

T
S

"My loop is wrong, right?"

"Use range(len(list)) here and += on line 12."

T
S

"Okay, got it."

The Result

Code works, but the student didn't learn why it was broken or how to find similar issues next time.

Debugging Literacy

Focus: Building thinking process

S

"Before I change anything, what should my code do?"

"Good question! What do you expect it to do?"

C
S

"Add up all the numbers in the list."

"And what's it actually doing instead?"

C
S

"Only giving me the last number."

"Walk me through what happens each loop iteration—focus on the total variable."

C
S

"Oh! I'm using = instead of +=. It overwrites instead of adding!"

The Result

Student discovered the bug themselves and built a mental model for finding similar issues independently.

The Key Difference

Debugging literacy isn't about teaching Python—it's about teaching students to slow down, form hypotheses, and test systematically. The goal is to build a thinking process they can use every time something breaks.
